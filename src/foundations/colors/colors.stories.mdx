# Color

> Color is a fundamental building block of any interface and is an integral part of how a product feels.
> Consistent application of color also helps users learn an interface.

## Palettes

### Primary Palette

Our Primary Palette is comprised of blue, indigo, and neutral tones to bring trust and boldness to our brand. We use
these colors in logical ways to guide the eye and highlight important information.

    <Swatch color={'#1E325C'} colorName={'Indigo'} shade={500} variant={'bottom'} />

## Semantic Colors

Semantic colors help users identify status, see actions, locate help, and understand the next steps they need to take.
The consistent use of color keeps the cognitive load low and makes for a unified and engaging user experience.

> **NOTE:** These colors are shades of the corresponding Secondary Palette above.

## Shades

Each of the Primary and Secondary colors have shades ranging from 100-800 with the exception of Neutrals which have a
wider range of shades. See below for details.

### Neutral Colors

Neutral colors have an increased level of variance to be used for text, elevations, and subtle backgrounds. We also use
neutrals when we don't want to draw too much attention to a particular touchpoint or convey information such as a
disabled option.

export const argTypes = {
    color: {
        control: null,
    },
};

export const Template = args => {
    const colors = createColorShades(args.color);
    return (
        <>
            <GlobalStyle />
            <Grid row>
                <Grid>
                    {colors &&
                        Object.keys(colors).map(shade => (
                            <Swatch
                                key={`color_${shade}`}
                                color={colors[shade]}
                                colorName={args.name || ''}
                                shade={shade}
                            />
                        ))}
                </Grid>
            </Grid>
        </>
    );
};
